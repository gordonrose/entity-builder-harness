# Scene Card: Coding Agent Work Actor

## Source

`reader-world-research-packet.md`

## What Happened

Public coding agents now behave like work actors. GitHub's coding agent can be
assigned a task, boot a virtual machine, clone a repository, save changes,
record reasoning in session logs, and tag a developer for review.

## Human Pressure

The developer wants the work to happen asynchronously, but still needs to know
where it happened, what changed, and what kind of review is required.

## Visible Objects

- virtual machine
- cloned repository
- draft pull request
- session log
- review tag

## Why It Matters

This scene bridges the author's local harness to the reader's public AI world.
It shows that the harness problem is not private eccentricity.
