# Improve Education System

## Use When

Use this when the user wants to incorporate world-class source material to
improve education output quality.

## Inputs

- Source description or provided source notes
- Target concern, such as teaching, storytelling, humor, talk design, writing
  style, or voice
- Current references, profiles, templates, and prompts

## Process

1. Identify the source and what quality problem it may improve.
2. Extract durable principles, not stylistic imitation.
3. Avoid copying source phrasing or copyrighted expression.
4. Decide where each principle belongs:
   - references for stable principles
   - profiles for taste and voice calibration
   - templates for output structure
   - prompts for task instructions
   - workflows for process changes
5. Record source review notes when external guidance affects the harness.

## Outputs

Potential targets include:

- `../references/teaching-principles.md`
- `../references/storytelling-principles.md`
- `../references/humor-principles.md`
- `../references/talk-design-principles.md`
- `../references/writing-style-principles.md`

## Quality Rules

- Use sources as a quality compass, not a ghostwriter.
- Keep generated outputs grounded in repo evidence and the author's voice.
- Do not ask the model to write like a specific living author, speaker, or
  comedian.
