# v0.1 Blog Draft Prompt

Draft the selected blog idea.

Inputs:

- Selected idea
- Relevant extracted material
- Voice profile
- Humor profile
- Storytelling profile
- Prior feedback

Write a 3-minute blog post of 450 to 650 words.

Requirements:

- first person
- clear enough for students
- credible enough for practitioners
- specific to the actual work
- lightly funny, not gag-heavy
- reflective but not pompous
- practical but not bland
- no corporate tone
- no fake epiphany
- no generic opening like "In software development..."

Use this structure:

1. A specific moment
2. The problem I thought I was solving
3. The problem I was actually solving
4. The lesson
5. A clean landing

After the draft, include:

- what works
- what may sound unlike me
- 3 alternate titles
- 3 possible stronger endings
- specificity audit
- voice audit
