# Shared Workflows

## Workflows

- `chat-start-interview.md` - identify active session metadata at chat start.
- `change-shared-process.md` - change shared chat, branch, git, commit, merge,
  handoff, deployment, release, or context-preservation process.
- `main-updated.md` - inspect and refresh chat branches after `main` changes.
- `local-convergence.md` - promote completed chat work into local `main`.
- `capability-resolution-workflow.md` - propose or resolve capabilities before
  implementation.
- `default.md` - placeholder retained for compatibility; do not use for real
  implementation work.
